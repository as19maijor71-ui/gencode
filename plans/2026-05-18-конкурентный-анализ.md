# План: Режим с конкурентным анализом (18.05.2026)

## Цель
Селлер копирует текст карточки конкурента → AI анализирует слабые места и выдаёт 6-е сообщение «🎯 Как обойти конкурента».

## Контекст
Вторая фича по чертежу `architecture/02-core/главный-цикл.md` (Режим B). Базовая генерация уже работает. **Без парсинга URL** — селлер копирует текст конкурента вручную. Без изменений в CardResult — конкурентный анализ отдельным вызовом AI.

## Критерий «фича готова»
5 товаров с текстом конкурента + 5 без = 10 генераций → у 5 с конкурентом 6-е сообщение содержит минимум 2 конкретные точки обхода со ссылкой на данные конкурента → у 5 без конкурента ничего не сломалось, CardResult не изменился.

## Задачи

### Фаза 1: Конкурентный анализатор (ядро)

- [x] **1.1 Функция analyze_competitor()**
  - `cardgen/engine/generator.py` — `analyze_competitor(my_description: str, my_category: str, competitor_text: str) -> str`
  - Отдельный вызов AI (не трогает CardResult, не трогает generate_card)
  - Промт: «Вот наша карточка. Вот карточка конкурента. Найди 3-5 слабых мест конкурента и дай конкретные точки обхода.»
  - Если AI упал → возвращает пустую строку (не блокирует основной flow)
  - **Критерий:** вызов возвращает строку длиной > 100 символов с нумерованными пунктами

- [x] **1.2 Валидация входного текста**
  - Обрезка текста конкурента до 3000 символов (перед отправкой в AI)
  - Минимум 50 символов (чтобы отсечь случайный мусор)
  - **Критерий:** `analyze_competitor("кружка", "home", "a" * 30)` возвращает пустую строку

### Фаза 2: Бот

- [x] **2.1 FSM — состояние competitor_input**
  - После подтверждения (`confirm:yes`) — не сразу генерация, а вопрос: «Хочешь анализ конкурента? Скопируй текст его карточки (заголовок + описание + характеристики) или нажми Пропустить.»
  - Кнопки: `[⏭ Пропустить]`
  - Новое состояние FSM: `competitor_input`
  - Валидация: не более 3000 символов, не менее 50 (если короче — «слишком мало, скопируй заголовок и описание»)
  - **Критерий:** flow с текстом → генерация + 6-е сообщение; flow с «Пропустить» → генерация и 5 сообщений как раньше

- [x] **2.2 Вывод 6-го сообщения**
  - `send_results()` — принимает опциональный `competitor_analysis: str = ""`
  - Если не пустой → 6-е сообщение «🎯 Как обойти конкурента»
  - **Критерий:** пустая строка → 5 сообщений; непустая → 6 сообщений, 6-е в HTML

### Фаза 3: Интеграция

- [x] **3.1 Сборка flow в хендлере**
  - `confirm:yes` → `competitor_input`
  - Пользователь вводит текст или жмёт «Пропустить»
  - «Пропустить» → генерация как раньше
  - Текст → параллельно: `generate_card()` + `analyze_competitor()` → 6 сообщений
  - **Критерий:** полный flow без крашей, оба пути работают

- [x] **3.2 Конфиг**
  - `config.py` — добавить `COMPETITOR_MAX_LENGTH: int = 3000`
  - `.env.example` — добавить строку
  - **Критерий:** настройка читается, значение по умолчанию 3000

## Риски

| Риск | Смягчение |
|---|---|
| AI не находит реальные точки обхода, пишет воду | Промт требует минимум 2 пункта со ссылкой на конкретные данные конкурента. Если пусто — не показываем 6-е сообщение |
| Селлер копирует страницу целиком (меню, футер, похожие товары) | Обрезка до 3000 символов; промт AI: «извлеки из текста только данные о товаре конкурента» |
| Конкурент из другой категории | Детектор категорий (уже есть) + предупреждение в competitor_input: «⚠️ Убедись, что конкурент из той же категории» |
| Двойной вызов AI = двойное время ожидания | Параллельный вызов `asyncio.gather(generate_card(), analyze_competitor())` — ждём одновременно, не последовательно |
