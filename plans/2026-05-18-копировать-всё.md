# План: Кнопка «Копировать всё» (18.05.2026)

## Цель
После генерации — одна кнопка «📋 Копировать всё», отправляющая единым текстом полную карточку для вставки на WB/Ozon.

## Контекст
Третья фича. Базовая генерация и конкурентный анализ работают. Селлер копирует блоки по одному (5-6 действий).

## Критерий «фича готова»
1. После `send_results()` появляется сообщение с кнопкой «📋 Копировать всё»
2. Нажатие → бот отправляет текст всей карточки чистым текстом, без HTML-тегов
3. Оба пути работают: с конкурентом (6 сообщений + кнопка) и без (5 сообщений + кнопка)
4. **Регрессия:** flow без нажатия кнопки не сломан — 5/6 сообщений как раньше
5. **Идемпотентность:** нажатие на старую кнопку (из истории чата, после новой генерации) → «Данные устарели. Сгенерируй заново.», не краш и не чужой текст

## Задачи (одна фаза)

- [x] **1. Функция `format_for_copy()`**
  - `cardgen/engine/generator.py` — `format_for_copy(result: CardResult, competitor_analysis: str = "") -> str`
  - Собирает все поля в чистый текст с секциями `===`, без HTML-тегов
  - **Критерий:** строка не пустая, содержит все секции, `<` и `>` отсутствуют

- [x] **2. `_split_by_sections()` — разбивка длинного текста**
  - Там же — `_split_by_sections(text: str, max_len: int = 4096) -> list[str]`
  - Разбивает по границам секций `===` (не рвёт посередине строки)
  - **Критерий:** каждый чанк ≤ 4096, каждый чанк начинается с `===` (кроме первого, если < 4096 и нет `===`)

- [x] **3. In-memory cache + кнопка**
  - `handlers.py` — словарь `_copy_cache: dict[str, str]`, функция `_store_copy(text: str) -> str` (возвращает ключ), `_get_copy(key: str) -> str | None` (pop)
  - `_do_generate()` — после `send_results` вызывает `format_for_copy`, сохраняет в cache, отправляет **отдельное** сообщение с инлайн-кнопкой «📋 Копировать всё» (callback_data = `copy:<key>` — короткий, ~10 байт)
  - **Критерий:** после результатов появляется новое сообщение с кнопкой; callback_data < 20 байт

- [x] **4. Хендлер кнопки**
  - `@router.callback_query(F.data.startswith("copy:"))` — без фильтра состояния (кнопка может быть нажата в любой момент)
  - Достаёт текст из `_get_copy(key)`. Если `None` → отвечает «⚠️ Данные устарели. Сгенерируй заново.»
  - Если текст есть → `_split_by_sections()`, отправляет частями как обычные сообщения (без HTML, моноширинный не нужен — чистый текст копируется долгим нажатием в Telegram)
  - Редактирует сообщение с кнопкой: убирает reply_markup (`edit_reply_markup(None)`)
  - **Критерий:** 1 нажатие → кнопка исчезает → чанки с текстом отправлены

## Риски

| Риск | Смягчение |
|---|---|
| Текст > 4096 символов | `_split_by_sections()` — разбивка по границам `===`, без разрыва строк |
| Старая кнопка из истории → чужой товар | `_get_copy()` — pop, ключ одноразовый. Повторное нажатие → «Данные устарели» |
| Утечка памяти в `_copy_cache` | pop при чтении. Неиспользованные ключи — ограничены кол-вом сессий, утечка ~100 байт/запись |

## Разбор критики (что взял, что нет)

| Критика | Вердикт | Обоснование |
|---|---|---|
| Две фазы — избыточно | **Взято** | Одна фаза, задача мелкая |
| Идемпотентность кнопки | **Взято** | In-memory cache с pop — ключ одноразовый |
| callback_data > 64 байт | **Взято** | Ключ ~10 байт вместо текста карточки |
| Регрессионный критерий | **Взято** | Добавлен в критерии |
| `<pre>` на мобильных | **Отклонено** | `<pre>` не используем — чистый текст копируется долгим нажатием на iOS/Android |
| Кнопка к какому сообщению | **Взято** | Отдельное сообщение, не edit существующего |
| Сценарий «старая кнопка» | **Взято** | Явная обработка `_get_copy → None` |
| Порог > 500 символов | **Отклонено** | Бессмысленно. Критерий: секции не пустые |
| Размер > 4096 без стратегии разбивки | **Взято** | `_split_by_sections` по границам `===` |
| `format_for_copy` в generator.py | **Отклонено** | 15 строк. Выносить в отдельный файл — over-engineering |
| `model_dump_json() → model_validate_json()` | **Взято** | Храним готовую строку, не модель |
| `copy_all_keyboard()` в keyboards.py | **Отклонено** | Кнопка строится на месте, одна строка |
| Хендлер читает стейт | **Взято** | Cache, не стейт |

## Итог

**Реализовано (18.05.2026):**

| Файл | Что |
|---|---|
| `engine/generator.py:228-304` | `format_for_copy()` — сборка всех полей CardResult в чистый текст с секциями `===`. `split_by_sections()` — разбивка по границам строк (не рвёт строку пополам), каждый чанк ≤ 4096 |
| `bot/handlers.py:27-37` | `_copy_cache` (dict), `_store_copy()`, `_get_copy()` — in-memory cache с pop (ключ одноразовый) |
| `bot/handlers.py:269-274` | В `_do_generate()`: после `send_results()` — формат текста → кеш → сообщение с инлайн-кнопкой «📋 Копировать всё» (callback_data = `copy:<key>`, ~10 байт) |
| `bot/handlers.py:296-308` | `copy_all` handler: без state filter, `_get_copy(key)` → `None` = «Данные устарели», иначе → `split_by_sections` → отправка чанками, кнопка исчезает |

**Flow:** генерация → 5-6 сообщений результатов → «📋 Нажми, чтобы скопировать...» с кнопкой → «✅ Готово». Нажатие: кнопка исчезает → бот отправляет полный текст карточки (разбитый на чанки если > 4096).

**Безопасность:** ключ одноразовый (pop) — повторное нажатие или старая кнопка из истории → «Данные устарели». Без HTML в выводе. Cache ограничен 1000 записей (FIFO-вытеснение).

**Правки по код-ревью (18.05.2026):**
1. `split_by_sections()` — переписан: группирует по границам `===`, затем по строкам, затем посимвольно
2. `copy_all` handler — `try/except` + `logger.warning` при ошибке отправки чанка
3. `_store_copy()` — лимит `_COPY_CACHE_MAX = 1000`, удаление старейшей записи при превышении
