# План: Первая фича — базовый режим Ai Маркетолог (17.05.2026)

## Цель
Работающий Telegram-бот: селлер описывает товар → AI-маркетолог генерирует полную стратегию карточки для WB и Ozon.

## Контекст
Первая реализация по чертежу `architecture/09-roadmap/первая-фича.md`. Базовый режим, без анализа конкурента по ссылке.

## Критерий «фича готова»
10 разных товаров → 10 успешных генераций → 5 сообщений-результатов → без ошибок и пустых полей.

## Риски

| Риск | Смягчение |
|---|---|
| OpenRouter API недоступен | YandexGPT fallback |
| JSON от AI невалиден | Повторный запрос, fallback — raw_response |
| Сообщение > 4096 символов | Разбивка на 5 сообщений |
| Изменение алгоритмов WB/Ozon | Модульная архитектура, замена 5 минут |

## Итог

### Реализовано
Все 8 задач трёх фаз выполнены. Код прошёл циклы из 5 ходов с исправлением найденных ошибок.

**Фаза 1 (Ядро):**
- `config.py` — pydantic-settings, 12 переменных, .env + defaults
- `wb_algorithm.py` / `ozon_algorithm.py` — правила площадок v2026.1
- `categories.py` — 7 категорий, 10 полей в каждой
- `prompts.py` — PROMPT_TEMPLATE + build_prompt() с подстановкой правил
- `generator.py` — CardResult (12 полей), call_openrouter→YandexGPT fallback, JSON-парсинг с retry

**Фаза 2 (Бот):**
- `keyboards.py` — InlineKeyboard (категории), ReplyKeyboard (начать заново)
- `handlers.py` — FSM (3 состояния), /start, валидация, HTML parse_mode, разбивка длинных сообщений
- `main.py` — polling, MemoryStorage

**Фаза 3 (Инфраструктура):**
- `Dockerfile` — python:3.11-slim
- `docker-compose.yml` — один сервис, restart: unless-stopped
- `.dockerignore` — исключены .env, .git, архитектура
- `.env.example` — все переменные с комментариями

### Исправлено в циклах 5 ходов
| Фаза | Ошибки |
|---|---|
| 1 | Fallback OpenRouter→YandexGPT, orphan MAX_RETRIES, unused imports |
| 2 | Хардкод категории, parse_mode, сообщения >4096, генерация без guard |
| 3 | Deprecated version, лишние tty/stdin, отсутствие .dockerignore |

### Осталось
- Заполнить `.env` реальными ключами (BOT_TOKEN, OPENROUTER_API_KEY)
- Протестировать 10 генераций на реальных товарах
- Критерий «фича готова» требует эмпирической проверки
