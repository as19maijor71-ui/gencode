## Что это

**Ai Маркетолог** — Telegram-бот. AI-маркетолог, который анализирует нишу селлера и создаёт продающую карточку для WB и Ozon. Подробности — `architecture/README.md`.

**Слоган:** Не текст. Стратегия.

## Стек

- **Язык:** Python 3.11+
- **Бот:** aiogram 3.4+
- **Валидация:** Pydantic 2.5+
- **AI:** OpenRouter (deepseek-r1) основной, YandexGPT fallback
- **Деплой:** Docker + docker-compose (polling, не webhook)

## 8 правил репо

1. **НИКАКОГО КОДА до проработки плана.** Сначала idea → architecture → plans, потом код.
2. **Не деплоить без моей явной команды.**
3. **Не коммитить без явной команды.**
4. **Не пушить без подтверждения.**
5. **Не менять idea/ и architecture/ без вопроса.** Это чертёж, не черновик.
6. **Большая фича = план в plans/.** Микрофича — без.
7. **Строгая типизация.** Все функции с аннотациями типов.
8. **.env только через config.py.** Никогда не читать .env напрямую.

## Большая фича = план

- Один план = одна фича
- Имя: `plans/YYYY-MM-DD-имя.md`
- Статусы: `[ ]` / `[WIP]` / `[x]`
- По задаче: цель / deliverable / критерий «сделано»
- В конце — итог: что реализовано, что осталось

## Цикл из 5 ходов после фичи

1. Найди ошибки → докажи → реши
2. Что упустили → докажи
3. Это РЕАЛЬНО проблема нашего проекта?
4. Свежий взгляд после правок
5. Security review (секреты, ввод пользователя, утечки)
