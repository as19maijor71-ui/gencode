# 07-decisions — Решения

## Технические решения

1. **Python вместо Node.js/Go** — быстрее прототипирование, aiogram — зрелая библиотека, один разработчик.

2. **Polling вместо webhook** — проще деплой, не нужен домен и SSL на старте. Минус: потребление ресурсов при масштабировании. Решаемо позже.

3. **MemoryStorage вместо БД** — для MVP без персистентности достаточно. В следующей версии — PostgreSQL.

4. **OpenRouter вместо прямого OpenAI** — дешевле, доступнее из РФ, `response_format: json_object`.

5. **Монолит вместо микросервисов** — весь код в одной кодовой базе. Один деплой. Один Docker-контейнер.

## Продуктовые решения

1. **Telegram-бот вместо веб-интерфейса** — быстрее разработка, нативный UX для селлеров (они уже в Telegram).

2. **Две стратегии (WB + Ozon) в одном ответе** — селлеры часто продают на обеих площадках. Не надо делать два запроса.

3. **Бесплатный пробный период** — снижает барьер входа. Конверсия в платный тариф через лимиты на количество генераций.

## Trade-offs

| Решение | Плюс | Минус | Когда пересмотреть |
|---|---|---|---|
| Polling | Простой деплой | Не масштабируется на 1000+ пользователей | При достижении 500 MAU |
| MemoryStorage | Нет зависимостей от БД | Потеря данных при перезапуске | При появлении платных подписок |
| OpenRouter | Дешёво, доступно | Качество ниже, чем у Claude напрямую | Если бюджет позволит перейти на Claude API |
| Без авторизации | Минимальный барьер входа | Нет защиты от злоупотреблений | При запуске платного тарифа |
