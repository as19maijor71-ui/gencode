# Что НЕ выбрали и почему

| Технология | Почему отказались |
|---|---|
| **TypeScript + Hono + Bun** | Предпочтение пользователя для веб-приложений, но для Telegram-бота Python/aiogram даёт: FSM из коробки, богатую экосистему, Pydantic-валидацию. Смена стека не оправдана для MVP. |
| **FastAPI / Flask** | Не нужен HTTP-сервер в MVP. Бот на polling. |
| **PostgreSQL / Redis** | MVP без персистентности. MemoryStorage для FSM. |
| **React / Next.js** | Веб-интерфейс — Фаза 3. |
| **OpenAI напрямую** | Дороже OpenRouter. Проблемы с доступом из РФ. |
| **Claude API** | Лучшее качество, но дороже и сложнее с доступом из РФ. |
| **Kandinsky / Flux** | Генерация изображений не в MVP. |
| **Kubernetes** | Один контейнер, один сервер. K8s — перегруз. |
| **Webhook** | Нужен домен + SSL. Polling проще для старта. |
| **Celery / RabbitMQ** | Нет очередей задач. Одна генерация = один запрос. |

## Когда пересмотреть

| Отказ | Когда вернуться | Триггер |
|---|---|---|
| TypeScript/Hono | Если делаем веб-интерфейс | Фаза 3 |
| PostgreSQL | При появлении пользователей и истории | 100+ пользователей |
| Webhook | При масштабировании | 500+ одновременных сессий |
| Claude API | При росте требований к качеству | Бюджет позволяет |
