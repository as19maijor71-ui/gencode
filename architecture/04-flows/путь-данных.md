# Путь данных

## Полный путь запроса

```
Селлер в Telegram
  │ описание товара (строка до 2000 символов)
  │ [+ URL конкурента — опционально]
  ▼
bot/handlers.py
  │ FSM: product_input → generating
  │ form_data = {category, description, competitor_url}
  ▼
engine/generator.py :: generate_card()
  │ 1. Получить список характеристик (templates/categories.py)
  │ 2. Построить промт (templates/prompts.py)
  │    - Вставить правила WB (engine/wb_algorithm.py)
  │    - Вставить правила Ozon (engine/ozon_algorithm.py)
  │    - Вставить описание товара
  │    - [Опционально] Извлечь данные из URL конкурента
  ▼
HTTP POST → OpenRouter API
  │ Headers: Authorization: Bearer {OPENROUTER_API_KEY}
  │ Body: model=deepseek-r1, max_tokens=4096, response_format=json_object
  │ [Fallback: YandexGPT]
  ▼
JSON-ответ от AI
  │ response["choices"][0]["message"]["content"]
  ▼
engine/generator.py :: parse_response()
  │ 1. Извлечь JSON (может быть в ```json ... ```)
  │ 2. Валидация через Pydantic CardResult
  │ 3. Если невалиден → повторный запрос (1 раз)
  │ 4. Если снова невалиден → raw_response, пустые поля
  ▼
bot/handlers.py
  │ Форматирование 5 сообщений
  │ Markdown → Telegram
  ▼
Селлер получает:
  ├─ 🟣 WB: заголовок, описание, ключи
  ├─ 🔵 Ozon: заголовок, описание, ключи
  ├─ 🎬 Сценарий видео Ozon
  ├─ 📋 Характеристики + 📸 Фото
  └─ 🎯 Стратегия «что делать прямо сейчас»
```

## Ключевые точки отказа

| Точка | Что может пойти не так | Резервный план |
|---|---|---|
| OpenRouter API | Таймаут, 5xx, нет ответа | YandexGPT |
| JSON невалиден | AI вернул текст вместо JSON | Повторный запрос с уточнением |
| Telegram API | Сообщение > 4096 символов | Разбивка на 5 отдельных сообщений |
| Ввод > 2000 символов | Селлер вставил простыню | Сообщение об ошибке, просьба сократить |
| URL конкурента недоступен | 404, таймаут | Игнорируем URL, генерируем в базовом режиме |
