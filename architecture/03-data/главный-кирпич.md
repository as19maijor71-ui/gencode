# Главный кирпич: CardResult

## CardResult — центральная сущность данных

Вся работа продукта крутится вокруг одной модели Pydantic:

```python
class CardResult(BaseModel):
    # === WB ===
    wb_title: str                      # 60-100 символов, CTR-оптимизированный
    wb_description: str                # продающий текст, 5-7 блоков, dwell-time
    wb_keywords: list[str]             # 10-15 LSI-ключей
    wb_photo_recommendations: str      # рекомендации по фото для WB

    # === Ozon ===
    ozon_title: str                    # до 150 символов, структурированный
    ozon_description: str              # сухой structured data текст
    ozon_keywords: list[str]           # 20-30 SEO-ключей
    ozon_video_script: str             # сценарий видео, 15-60 сек
    ozon_photo_recommendations: str    # рекомендации по фото для Ozon

    # === Общее ===
    characteristics: dict[str, str]    # все поля характеристик
    strategy_notes: str                # 3-5 пунктов: «что делать прямо сейчас»

    # === Служебное ===
    raw_response: str = ""             # сырой ответ AI (для отладки)
```

## Почему это главный кирпич

1. **Всё приложение — это генерация одного CardResult.** Нет других сущностей данных.
2. **CardResult = контракт с AI.** AI обязан вернуть JSON именно этой структуры.
3. **CardResult = интерфейс.** Бот разбирает поля модели на 5 сообщений.

## Что точно НЕ входит в CardResult

- Данные пользователя (нет авторизации в MVP)
- История генераций
- Изображения (только текст)
- Аналитика (цифры продаж)
