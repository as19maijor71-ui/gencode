# Первая фича

## Что строим

**Базовый режим Ai Маркетолог** — Telegram-бот, принимающий описание товара и возвращающий полную стратегию карточки.

## Deliverable

Работающий Telegram-бот с одним полным циклом:
- `/start` → выбор категории → ввод описания → генерация → 5 сообщений-результатов

## Что входит

| Компонент | Файл |
|---|---|
| Конфигурация | `cardgen/config.py` |
| Зависимости | `cardgen/requirements.txt` |
| Правила WB | `cardgen/engine/wb_algorithm.py` |
| Правила Ozon | `cardgen/engine/ozon_algorithm.py` |
| Категории | `cardgen/templates/categories.py` |
| Промт | `cardgen/templates/prompts.py` |
| Генератор | `cardgen/engine/generator.py` (CardResult, call_ai, parse_response, generate_card) |
| Клавиатуры | `cardgen/bot/keyboards.py` |
| Хендлеры | `cardgen/bot/handlers.py` (FSM, команды, вывод) |
| Точка входа | `cardgen/bot/main.py` |
| Деплой | `Dockerfile`, `docker-compose.yml`, `.env.example` |

## Что НЕ входит

- Режим с конкурентным анализом (ссылка)
- Кнопка «Копировать всё»
- Авторизация
- Ограничения по количеству генераций

## Критерий готовности

```
1. Запуск: docker compose up → бот отвечает на /start
2. Выбор категории → клавиатура работает
3. Ввод описания → бот сообщает «⏳ Анализирую...»
4. Через ~30 секунд → 5 сообщений с результатами
5. Все поля CardResult заполнены (нет пустых строк)
6. 10 разных товаров → 10 успешных генераций без ошибок
```

## Порядок реализации

1. `config.py` + `requirements.txt` + `pyproject.toml`
2. `engine/wb_algorithm.py` + `engine/ozon_algorithm.py`
3. `templates/categories.py` + `templates/prompts.py`
4. `engine/generator.py` (CardResult, вызов AI, парсинг)
5. `bot/keyboards.py` → `bot/handlers.py` → `bot/main.py`
6. `.env.example` + `Dockerfile` + `docker-compose.yml`
