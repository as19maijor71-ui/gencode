# Границы системы

## Что внутри (наша ответственность)

- Telegram-бот и его логика
- Построение промта для AI
- Вызов OpenRouter / YandexGPT
- Парсинг и валидация AI-ответов
- Файлы правил алгоритмов WB/Ozon
- Шаблоны категорий и характеристик

## Что снаружи (используем как сервис, доверяем)

- **OpenRouter API** — генерация текста. Доверяем в рамках качества ответа, НЕ доверяем в вопросах безопасности (ключи, лимиты).
- **YandexGPT API** — fallback-провайдер. Доверяем так же, как OpenRouter.
- **Telegram API** — доставка сообщений. Доверяем как транспорту.

## Граница доверия

| Источник | Уровень доверия | Почему |
|---|---|---|
| **Ввод селлера** (описание товара) | 🔴 Не доверяем | Может содержать инъекции, спам, попытки манипуляции AI |
| **Ссылка на конкурента** (URL) | 🔴 Не доверяем | Может вести на фишинг, вредоносный контент |
| **OpenRouter ответ** | 🟡 Частично доверяем | Валидируем через Pydantic. Может быть невалидным JSON |
| **YandexGPT ответ** | 🟡 Частично доверяем | Валидируем. Может не поддерживать JSON-формат |
| **Собственный код** | 🟢 Полностью доверяем | Контролируем |

## Один сервер, одна база, один деплой

- Один Docker-контейнер
- Без микросервисов
- Без Kubernetes
- Пока физически не невозможно — не усложняем
