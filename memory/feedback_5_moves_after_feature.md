# feedback_5_moves_after_feature

**type:** feedback

**Правило:** После завершения фичи — цикл из 5 ходов проверки перед тем, как считать фичу готовой.

**5 ходов:**
1. Найди ошибки → докажи → реши
2. Что упустили → докажи
3. Это РЕАЛЬНО проблема нашего проекта?
4. Свежий взгляд после правок
5. Security review (секреты, ввод пользователя, утечки)

**Why:** Первая версия всегда содержит пропуски. Цикл ловит их системно, а не по наитию.

**How to apply:** После того как «код написан и работает» — не останавливайся. Пройди 5 шагов последовательно, документируй находки.
