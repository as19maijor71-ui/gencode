# feedback_no_code_without_plan

**type:** feedback

**Правило:** Никакого кода до полной проработки и утверждения плана реализации.

**Why:** Опыт проекта: код, написанный без плана, переписывается 2-3 раза. Сначала idea → architecture → plans → потом код.

**How to apply:** При любой задаче на код — сначала проверь, есть ли план в plans/. Если нет — создай, утверди, потом пиши код.
